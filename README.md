JavaScript - bu World Wide Webning asosiy texnologiyasi :
JavaScript - bu yuqori darajadagi dasturlash tili bo'lib, u veb-sahifalar harakati uchun mijoz tomonidan qo'llaniladi.
Bu HTML va CSS bilan bir qatorda World Wide Webning asosiy texnologiyalaridan biridir.
Barcha yirik veb-brauzerlarda foydalanuvchilarning qurilmalarida kodni bajarish uchun maxsus JavaScript dvigateli mavjud.
JavaScript ko'p paradigmali tildir :
JavaScript - bu ECMAScript standartiga mos keladigan yuqori darajali, ko'pincha o'z vaqtida kompilyatsiya qilingan til.
Bu ko'p paradigma bo'lib, voqealarga asoslangan, funktsional va imperativ dasturlash uslublarini qo'llab-quvvatlaydi.
U matn, sanalar, muntazam ifodalar, standart ma'lumotlar tuzilmalari va Hujjat Ob'ekt Modeli (DOM) bilan ishlash uchun amaliy dasturlash interfeyslariga (API) ega.
JavaScript dvigatellari serverlar va ilovalarda qo'llaniladi :
JavaScript dvigatellari dastlab faqat veb-brauzerlarda ishlatilgan, ammo hozirda ba'zi serverlar va turli xil ilovalarning asosiy komponentlari hisoblanadi.
Ushbu foydalanish uchun eng mashhur ish vaqti tizimi Node.js hisoblanadi.
JavaScript murakkab tarixga ega :
JavaScript 1995 yilda Netscape-da Brendan Eich tomonidan yaratilgan.
Microsoft o'zining JScript deb nomlangan JavaScript versiyasini yaratdi.
Netscape JavaScript-ni Ecma International-ga taqdim etdi, bu 1997 yil iyun oyida birinchi ECMAScript tili spetsifikatsiyasining rasmiy nashriga olib keldi.
Standartlar jarayoni bir necha yil davom etdi, ECMAScript 2 1998 yil iyun oyida va ECMAScript 3 1999 yil dekabrda chiqarildi.
ECMAScript 4 ustida ish 2000-yilda boshlangan, biroq u oxir-oqibatda taqlid qilingan.
ECMAScript 5 standarti 2009-yil dekabrda, ECMAScript 6 esa 2015-yilda chiqarilgan.
JavaScript veb-brauzerlardan tashqarida keng qo'llaniladi :
2009-yilda Node.js-ning yaratilishi JavaScript-dan veb-brauzerlardan tashqarida foydalanishning sezilarli darajada oshishiga olib keldi.
2018 yil holatiga ko'ra, Node millionlab ishlab chiquvchilar tomonidan ishlatilgan va npm dunyodagi har qanday paket menejerlarining eng ko'p modullariga ega edi.
Hozirgi JavaScript ekotizimida ko'plab kutubxonalar va ramkalar, o'rnatilgan dasturlash amaliyotlari va veb-brauzerlardan tashqari JavaScript-dan sezilarli darajada foydalanish mavjud.
JavaScript turli xil skriptli xatti-harakatlar uchun ishlatiladi :
JavaScript sahifani qayta yuklamasdan yangi veb-sahifa mazmunini yuklash, veb-sahifa animatsiyalari, brauzer o'yinlarini o'ynash, oqimli media ijrosini boshqarish, qalqib chiquvchi reklamalar yoki ogohlantirish qutilarini yaratish, ma'lumotlar yuborilishidan oldin veb-shaklning kirish qiymatlarini tekshirish uchun ishlatiladi. veb-serverga, foydalanuvchining xatti-harakatlari haqidagi ma'lumotlarni jurnalga yozib, keyin uni serverga yuborish, foydalanuvchini boshqa sahifaga yo'naltirish va foydalanuvchi qurilmasida ma'lumotlarni saqlash va olish.

Xususiyatlari :

JavaScript ish vaqtida satrlar sifatida taqdim etilgan bayonotlarni bajarishi mumkin bo'lgan baholash funktsiyasini o'z ichiga oladi.

Turli xil :
JavaScript odatda skriptlar atrof-muhit bilan o'zaro aloqada bo'lishi mumkin bo'lgan ob'ektlar va usullarni ta'minlash uchun ish vaqti muhitiga tayanadi.
JavaScript navbatdagi xabarlarni birma-bir qayta ishlaydi.
JavaScript har bir yangi xabar bilan bog'langan funktsiyani chaqiradi va funktsiya argumentlari va mahalliy o'zgaruvchilar bilan qo'ng'iroqlar to'plami ramkasini yaratadi.
Qo'ng'iroqlar to'plami funksiya ehtiyojlariga qarab qisqaradi va o'sadi.
Funktsiya tugagandan so'ng qo'ng'iroqlar to'plami bo'sh bo'lsa, JavaScript navbatdagi xabarga o'tadi.

JavaScript xavfsizligi :
JavaScript va DOM dan zararli mualliflar skriptlarni Internet orqali mijoz kompyuterida ishga tushirish uchun ishlatishi mumkin.
Brauzerlar bu xavfni ikkita cheklov yordamida kamaytiradi: skriptlar sinov muhitida ishlaydi va bir xil manba siyosati bilan cheklanadi.
ADsafe va Secure ECMAScript (SES) kabi yuqori darajadagi xavfsizlikni ta'minlaydigan JavaScript-ning quyi to'plamlari mavjud.
Saytlararo zaifliklar :
Saytlararo skript (XSS) JavaScript bilan bog'liq keng tarqalgan xavfsizlik muammosi bo'lib, tajovuzkor maqsadli veb-saytga qurbonga taqdim etilgan veb-sahifaga zararli skriptni qo'shishi mumkin bo'lganda yuzaga keladi.
Saytlararo so'rovlarni qalbakilashtirish (CSRF) - bu boshqa saytlararo zaiflik bo'lib, unda tajovuzkor saytidagi kod jabrlanuvchining brauzerini foydalanuvchi maqsadli saytda niyat qilmagan harakatlarni amalga oshirish uchun aldaydi.
Mijoz-server ilovalarini ishlab chiquvchilar ishonchsiz mijozlar tajovuzkorlar nazorati ostida bo'lishi mumkinligini tan olishlari va ularning JavaScript kodi mo'ljallangan tarzda ishlaydi deb o'ylamasliklari kerak.
Mijozga noto'g'ri ishonch :
Veb-sayt mualliflari JavaScript qanday ishlashini mukammal yashira olmaydi, chunki xom manba kodi mijozga yuborilishi kerak.
JavaScript shaklini tekshirish faqat xavfsizlikni emas, balki foydalanuvchilar uchun qulaylikni ta'minlaydi.
Skriptlarni tanlab o'chirib qo'yish mumkin, shuning uchun sichqonchaning o'ng tugmasi kabi operatsiyalarni oldini olish uchun JavaScript-ga tayanib bo'lmaydi.
